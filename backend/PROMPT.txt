**UdayaTech Design System Redesign Specification**  
**Enhanced Edition – Dawn Commerce: A Bold New Era of Warm, Stylish, and Premium Elegance**  

This transformative redesign doesn’t just update the interface — it unleashes a completely fresh, stylish, and sophisticated new look and feel that radiates confidence, warmth, and modern luxury. Every pixel has been reimagined to deliver an immersive, premium experience that feels instantly elevated, contemporary, and unmistakably UdayaTech.

1. Brand Concept: "Dawn Commerce"  
The name "Udaya" means sunrise in Sanskrit. The redesign leans into this origin to create a visual identity that is warm, emergent, and clear, standing apart from the cool-blue enterprise aesthetic of Indian ecommerce incumbents.  

**New stylish elevation:** Warm Authority — Deep teal-green primary replaces generic indigo/blue. Conveys trust and groundedness without imitating any major Indian ecommerce brand — now radiating a refined, premium depth that feels both calming and commanding.  
Sunrise Energy — A burnished amber accent drives CTAs and deal highlights. Amber is distinct from Flipkart's yellow and Amazon's orange; it reads as premium warmth — injecting vibrant, luxurious energy into every interaction.  
Service-First Identity — Service offerings get equal visual weight to products. The design system treats "what we install" with the same card anatomy as "what we sell," reinforcing the dual-commerce model — now with a harmonious, stylish balance that feels seamless and intentional.  
Breathing Room — Generous whitespace, larger touch targets, and a 6px base grid create a calm, confident rhythm that signals quality over density — delivering an airy, sophisticated spaciousness that invites exploration.  
Typographic Voice — Plus Jakarta Sans for headings (geometric, modern, has character) paired with Inter for body text creates a distinctive but readable hierarchy — now giving the entire interface a crisp, contemporary personality.  
Soft Geometry — A consistent 12px base radius for containers and a subtle brand-tinted shadow system replace the current mix of rounded-md/lg/xl/full. The shape language is "soft square," not pill-shaped, not sharp-cornered — crafting elegant, refined forms that feel modern and approachable.  
Unified System — Customer and admin surfaces share one token set, one radius, one shadow scale, and one type ramp. Admin simply uses a darker surface palette from the same hue family — creating a cohesive, stylish ecosystem across every touchpoint.

5. Accessibility Rules
5.1 Color Contrast
All text on --surface-card (white) must meet WCAG 2.1 AA (4.5:1 for normal text, 3:1 for large text)
--brand-600 (
#0A7E72) on white: contrast ratio ~5.2:1 (passes AA)
--accent-500 (
#D97706) on white: contrast ratio ~3.8:1 (passes AA for large text only; use --accent-600 for small text on white)
--text-tertiary (
#A1A1AA) on white: ~3.5:1 (use only for non-essential decorative text; ensure critical labels use --text-secondary minimum)
5.2 Focus Management
All interactive elements must show a visible focus ring: 2px solid var(--brand-500), outline-offset: 2px
For dark admin surfaces: focus ring 2px solid var(--admin-accent) (amber on dark teal)
Skip-to-content link at the top of CustomerLayout and AdminLayout
Modal focus trap: focus must cycle within the modal when open
5.3 Motion
All animations respect prefers-reduced-motion: reduce
Under reduced motion: disable transform transitions, carousel auto-play, and skeleton shimmer
Implementation in CSS:
@media (prefers-reduced-motion: reduce) {
  *, *::before, *::after {
    animation-duration: 0.01ms !important;
    transition-duration: 0.01ms !important;
    scroll-behavior: auto !important;
  }
}
5.4 Touch Targets
Minimum 44px x 44px for all interactive elements (buttons, links, nav items)
BottomNav items: minimum 48px wide
Input height: 44px minimum
5.5 Screen Reader
All icon-only buttons must have aria-label
Images must have meaningful alt text (product name, not "image")
Status badges should include role="status" for order status changes
Form errors should be linked via aria-describedby
